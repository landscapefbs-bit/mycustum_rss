# My Custom RSS Reader

自分専用にカスタマイズされた、AIアシスタント機能搭載の高機能RSSリーダーです。
フロントエンドはReact + Tailwind CSS、バックエンドはGo言語 (Gin + GORM) で構築されています。

## ✨ 主な機能

* **モダンでレスポンシブなUI**
  * ダークモード対応。
  * 3ペインレイアウト（サイドバー、記事一覧、詳細ビューア）。
* **AIによる記事の要約とタグ付け**
  * Gemini 2.5 Flash-Lite APIを活用し、取得した記事の要約（サマリー）とカテゴリタグを自動で生成します。
* **強力な検索とフィルタリング機能**
  * キーワード検索（記事本文からの検索）、タグによる絞り込み。
  * **自然言語での日時検索**: 「今日」「昨日」「1週間前から3日前まで」などの柔軟な日付指定が可能。システムが解析できない書き方の場合は、Gemini APIを用いて自動解析し、結果を学習・キャッシュします。
* **リッチコンテンツ表示**
  * `go-readability` と `html-to-markdown` を用いて、記事の本文を抽出しMarkdownとして綺麗に表示します。
  * Yahoo!ニュースのようなポータルサイトへリダイレクトされるフィードでも、本文を正しく取得する独自ロジックを搭載しています。
* **フィードの高度な管理機能**
  * **自動取得間隔**: フィードごとに更新間隔（15分〜24時間、または手動）を設定可能。
  * **残存期間とクリーンアップ**: 記事の保存期間を設定でき、古い記事は自動で削除されます。
  * ドラッグ＆ドロップによる直感的な並び替え。
* **お気に入り・後で読む**
  * 気になる記事をストック可能。「後で読む」などに登録された記事は、残存期間を過ぎても保護されます。
* **カスタマイズ可能な共有テンプレート**
  * 記事のタイトル、URL、配信日時などを変数として組み込み、SlackやSNSなどに一発で共有できるテンプレートを作成できます。

## 🚀 環境構築と起動方法

### 前提条件
* Go 1.21 以上
* Node.js 18 以上
* Gemini API キー（Google AI Studioから取得）

### 1. リポジトリのクローンと環境変数の設定
```bash
git clone https://github.com/landscapefbs-bit/mycustum_rss.git
cd mycustum_rss
```
`backend` フォルダ直下に `.env` ファイルを作成し、GeminiのAPIキーを設定してください。
```env
GEMINI_API_KEY=あなたのAPIキーをここに入力
```

### 2. バックエンドの起動
```bash
cd backend
go mod tidy
go run main.go
```
※ バックエンドは `http://localhost:8080` で起動し、SQLiteデータベース (`myrss.db`) が自動作成されます。

### 3. フロントエンドの起動
別のターミナルを開き、以下のコマンドを実行します。
```bash
cd frontend
npm install
npm run dev
```
※ フロントエンドは `http://localhost:5173` で起動します。ブラウザでアクセスしてください。

## 📖 操作方法

1. **フィードの追加**: サイドバーの「購読中」横にある「＋」ボタンからRSSのURLを入力して追加します。
2. **フィードの管理**: サイドバーの歯車アイコンから「フィード管理画面」を開き、自動取得間隔や残存期間の設定、ドラッグ＆ドロップでの並び替えが行えます。
3. **記事を読む**: フィードを選択し、中央のリストから記事をクリックすると右側に詳細が表示されます。未読記事には「NEW」バッジが付きます。
4. **検索する**: 記事一覧の上部にある「スライダー」アイコンをクリックすると、検索メニュー（キーワード、タグ、日時）が開きます。
5. **共有テンプレート**: サイドバー下部の歯車アイコン（設定）から、共有メッセージのフォーマットを自由に作成・編集できます。

## ⚠️ 制限事項・注意事項

* **Gemini APIのレート制限**: 
  * AIの要約処理には無料枠のAPIを利用することを想定しているため、1分間あたり15リクエストの上限（Rate Limit）に引っかからないよう、内部で「1記事あたり4秒の待機時間」を設けて直列処理しています。大量の新着記事を一気に取得した場合、すべての要約が完了するまでに少し時間がかかります。
* **ローカルデータベース**: 
  * 記事や設定のデータはすべて `backend/myrss.db` (SQLite) に保存されます。マルチユーザー対応のシステムではなく、あくまで個人利用（セルフホスト）を目的とした設計になっています。
* **自動取得**:
  * バックエンドサーバー（Go）が起動している間のみ、設定した間隔に従ってバックグラウンドで新しい記事を取得・クリーンアップします。
